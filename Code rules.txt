Принципы правильного кода:
1. Осмысленные названия для классов, методов и переменных

2. Документировать код. JavaDoc

3. Использовать this, super

4. Избегать непонятных констант (антипаттерн magic numbers)

5. Писать тесты на весь код

7. Использовать неизменяемые объекты, Immutable.

8. Всегда возвращать объекты. Избегать возвращать null

9. Использовать композицию вместо наследования (шаблон Декторатор).

10. Количество строк в методе не должно превышать 10 штук

11. Использовать один return на весь метод
